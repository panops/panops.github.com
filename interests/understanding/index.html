Purpose and Rationale

The PanOps Platform is designed to be a reliable, operable, distributed, self-organizing, resource management platform for distributed applications.

Reliable

Reliability is the prime aspiration and guarantee of PanOps. Reliability, for PanOps, is best described as determinism. Given a known state of those resources, and a transformation from that state to another given state PanOps components should always converge to the same result.

Operable

Operability refers to the ability of a system to keep its operators informed of its state in the past, present, and when known, future and to hold that state given changes in the surrounding resources. PanOps components chronicle all actions, decisions, and considerations and their reasons in a standard format using a standard interface for retrieval so that if human intervention into system operation is required to debug a problem or correct an issue context may be quickly gathered and corrective action may be taken.
